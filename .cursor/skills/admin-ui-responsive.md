# Cursor Skill: Admin UI + Responsive Pattern

## Use for
- admin page UI changes
- tables, cards, filters, mobile layouts
- theme/token updates

## Mandatory Docs
- `docs/DESIGN.md`
- `docs/ARCHITECTURE.md`

## Steps
1. Plan desktop + mobile behavior first.
2. Reuse existing components/patterns.
3. Keep numeric scanability and table clarity.
4. Implement mobile restructure for data-heavy views.
5. Validate accessibility and action reachability.
6. Verify dark/light token compatibility.

## Required Output
- desktop behavior summary
- mobile behavior summary
- theme compatibility note
- accessibility note
